The error occurred when I tried to filter the dataset to select individuals aged 75+ who were victims of crime. This happened because the variable agegrp7 was stored as "haven_labelled" (a labelled double type), while bcsvictim was stored as a factor. Since filter() requires both values being compared to have the same data type, the mismatch caused the operation to fail.

To fix this, I need to convert agegrp7 to a character or factor before filtering. In the future, I will be more aware that when working with survey data, especially imported from SPSS using haven, I should check variable types before applying filters. I will also ensure that labelled variables are converted properly using as_factor() or as.character() to prevent type mismatches.
